# Book Collection Manager

This project is a simple book collection management API built with FastAPI and SQLite. It provides a lightweight backend for storing, retrieving, and managing book records through RESTful endpoints.

The application uses SQLite as its database, making it easy to set up and run without additional configuration. FastAPI is used to handle HTTP requests, input validation, and automatic API documentation.

## Features

Create, read, update, and delete book records

RESTful API design

SQLite database for persistent storage

Automatic interactive API documentation via FastAPI (Swagger UI)

## Tech Stack

FastAPI – Web framework

SQLModel and SQLite – Database

Python – Programming language

## Purpose

This project is intended for learning and demonstration purposes, showcasing how to build a simple backend API using FastAPI with a relational database.

## Prerequisites
FastAPI Application requires FastAPI to be installed on your system in order to run. You can download and install the latest version of FastAPI from the official website (https://fastapi.tiangolo.com/).

## Installation
clone repository:
git clone https://github.com/452init/Book_collection_manager_api.git

navigate to project directory:
cd Book_collection_manager_api

install dependencies in a virtual environment then run fastapi dev main.py for development server
